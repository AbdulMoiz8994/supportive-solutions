Both core modules are built, and every dev-side item is now done and tested. The 3 AI buttons (document recognition, client Daily-brief, caregiver summary) were wired on 06/28 — nothing on our side is outstanding for these two modules. What remains is Agency action only: turn on the outside accounts in §4, after which the account-gated features (EVV, billing, background checks, AI email intake) come online in a few days each. Three of those portals (CHAMPS/MILogin, MDHHS, Sigma, ICHAT) have no public API at all, so they are automated by RPA (stored login) rather than a clean key.

✓ Quality: 100+ automated tests across the two modules' scope (AI edge cases — refusals, rate-limits, errors, bad input; backend automations; mobile clock-in API; billing/EVV status), out of 125+ test files in the suite. The three newly-wired AI buttons are covered by a fresh render/wiring test, and the regression set (AI · Clients · Caregivers · Documents = 70 tests) is green. Browser end-to-end checks pass 16/16. The Claude AI integration was verified live against the real service.
One compliance gate before going live on real data: the AI scan/summary features send protected health information (ID images, names, DOB, Medicaid IDs) to the AI provider. The agency must have a signed Business Associate Agreement (BAA) with Anthropic (or run Claude through the already-BAA'd AWS Bedrock). All features are built and tested regardless — this is only the switch for real client data.
